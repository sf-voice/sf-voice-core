**We make voice AI better.**

Our first feature is a search engine for media. Ingest audio or video, search it with natural language, and use the returned timestamps to jump back into the recording. You do not need to build your own transcript store, embedding pipeline, or media search layer first.

<Warning>
  These docs cover version `0.1.1`. We are iterating quickly, so pin your SDK version and expect breaking changes between versions until the API settles.
</Warning>

Search is where we start. Next up is benchmarking and toolings for voice AI systems.

## What you can build with our media SDKs. 

- Search all audio by topic, objection, or feature request as humanly possible **"Where does customers got impatient?"**.
- Find calls where your AI interrupted the caller, spoke too slowly, or repeated itself.
- Scope each end customer to their own audio.
- Build compliance review, QA workflows, or knowledge bases on top of your call archive.

## The core model

Assets carry IDs and scope from your own system.

| Field | What it is |
| --- | --- |
| `asset_id` | Your ID for the recording. Required. Returned on search results so you can join back to your system. |
| `asset_class` | A logical group — one customer, one workspace, one project. Scopes search so results don't leak across boundaries. |
| `types` | What to index: `audio`, `video`, `transcript`, or any combination. |
| `metadata` | Flat key/value pairs for correlation and display. |

## Lifecycle

<Steps>
  <Step title="Ingest">
    Send a URL, authenticate us to your bucket via S3 key, or file upload with your `asset_id`. The API returns a `task_id` immediately.
  </Step>
  <Step title="Index">
    Poll the task until it reaches `ready`. Indexing runs in the background.
  </Step>
  <Step title="Search">
    Query with natural language, scoped to an `asset_class` or explicit `asset_ids`. Results include `start_ms` and `end_ms`.
  </Step>
</Steps>

## Data and privacy

We index your media and keep the searchable representation: embeddings, transcripts, and metadata. Source files pass through the processing pipeline and are not retained after indexing completes.

**You keep ownership of your media. We store the indexed data needed for search.**

Regional processing:
- **US customers** — processed in the United States.
- **EU and UK customers** — processed in Europe.
