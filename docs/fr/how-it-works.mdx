## Les assets démarrent dans votre système

Les requêtes d'ingestion transportent votre `asset_id`, afin que les résultats de recherche puissent être reliés à votre base de données, CRM, ticket de support, plateforme d'observabilité ou journal d'appels, sans traduction d'identifiants.

```ts
await client.ingest({
  source: "url",
  asset_id: "call_8294",
  asset_class: "customer_acme",
  url: "https://storage.example.com/calls/8294.mp3",
});
```

## Les asset classes définissent la portée de la recherche

`asset_class` est la manière dont vous tracez la frontière de ce qui est recherché ensemble. Pour la plupart des produits, cela correspond à l'entité dans laquelle vos clients raisonnent :

- un client final
- un workspace
- un projet
- une file de support

<Warning>
  Si vous construisez une recherche destinée aux clients, limitez-la par `asset_class` ou par `asset_ids` explicites. La recherche globale (`scope: "all"`) parcourt l'ensemble de vos assets indexés.
</Warning>

## Types de surfaces d'indexation

`types` contrôle ce qui est indexé et recherché.

| Type | À utiliser pour |
| --- | --- |
| `audio` | Identité du locuteur, événements acoustiques, motifs au niveau du son. |
| `video` | Contenu à l'écran, contexte visuel, enregistrements d'écran. |
| `transcript` | Mots prononcés, récupération thématique, recherche sémantique sur les dialogues. |

La plupart des cas d'usage d'IA vocale veulent les trois :

```ts
types: ["audio", "video", "transcript"]
```

## L'indexation est asynchrone

L'ingestion renvoie immédiatement un `task_id`. L'indexation réelle se fait en arrière-plan. Interrogez jusqu'à `ready` ou `failed`.

```ts
const { task_id } = await client.ingest(request);
const task = await client.pollTask(task_id);

if (task.status === "failed") {
  throw new Error(task.error ?? "indexing failed");
}
```

États des tâches : `pending` → `indexing` → `ready` (ou `failed`).

## La recherche renvoie des correspondances horodatées

Les résultats incluent `start_ms` et `end_ms` : des décalages en millisecondes dans l'enregistrement source.

```json
{
  "asset_id": "call_8294",
  "score": 0.87,
  "start_ms": 42000,
  "end_ms": 58000,
  "match_type": "transcript"
}
```

`threshold` contrôle le niveau de stricte exigence d'une correspondance. La valeur par défaut est `0.5`. Augmentez-la pour obtenir moins de résultats, mais avec une plus grande confiance.

## Vos données restent les vôtres

L'API renvoie : `asset_id`, `asset_class`, `types`, scores et horodatages. Les identifiants des fournisseurs, les clés d'index et les détails du pipeline restent derrière l'API.
