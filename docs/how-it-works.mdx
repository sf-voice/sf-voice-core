---

sf-voice media has one core loop: submit an asset, wait for indexing, then
search the indexed media with the same ids and scope your application already
uses.

```mermaid
flowchart LR
  A[Your app] --> B[Ingest media]
  B --> C[Index video, audio, transcript]
  C --> D[Task becomes ready]
  D --> E[Search by asset_class or asset_ids]
  E --> F[Timestamped results]
```

## 1. Assets start in your system

Every ingest request includes your `asset_id`. This keeps the media API easy to
join back to your database, CRM, support ticket, customer workspace, or internal
object model.

```ts
await client.ingest({
  source: "url",
  asset_id: "video_123",
  asset_class: "customer_acme",
  url: "https://example.com/recording.mp4",
});
```

## 2. Asset classes define search scope

`asset_class` is the public grouping primitive. Use it for the boundary your
product cares about most.

Good examples:

- one end customer
- one workspace
- one project
- one repository
- one collection of support calls

<Warning>
  If you build customer-facing search, prefer `asset_class` or explicit `asset_ids`.
  Global search should be an explicit product decision.
</Warning>

## 3. Types choose searchable surfaces

`types` controls which surfaces you index or search.

| Type | Use it when |
| --- | --- |
| `video` | The visual content matters. |
| `audio` | Sound, speakers, or acoustic cues matter. |
| `transcript` | Spoken words and text retrieval matter. |

You can combine types:

```ts
types: ["video", "audio", "transcript"]
```

## 4. Indexing is asynchronous

Ingest returns quickly with a `task_id`. The asset is searchable after the task
reaches `ready`.

```ts
const ingest = await client.ingest(request);
const task = await client.pollTask(ingest.task_id);
```

Task polling returns the asset id, asset class, indexed types, status, and any
error if indexing fails.

## 5. Search returns timestamped matches

Search results include `start_ms` and `end_ms` so your UI can jump directly to
the matching moment.

```json
{
  "asset_id": "video_123",
  "score": 0.84,
  "start_ms": 42000,
  "end_ms": 58000,
  "match_type": "transcript"
}
```

Use `threshold` to control strictness. Higher values return fewer, more
confident matches.

## 6. Backend provider details stay hidden

The SDK only exposes sf-voice concepts:

- `asset_id`
- `asset_class`
- `types`
- `threshold`

Provider indexes, provider ids, and type mappings are handled by the backend.
