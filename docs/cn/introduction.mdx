San Francisco Voice Company 为语音 AI 产品构建媒体搜索 API 和 SDK。

我们的第一个功能是面向媒体的搜索引擎。摄入音频或视频，使用自然语言进行搜索，并使用返回的时间戳跳回到录音中。你无需先构建自己的转写存储、嵌入向量管道或媒体搜索层。

<Warning>
  本文档对应版本 `0.1.1`。我们迭代很快，请固定你的 SDK 版本，并预期在 API 稳定之前各版本间会出现破坏性变更。
</Warning>

搜索是我们的起点。接下来是面向语音 AI 系统的基准测试和调试工具。

## 你可以构建什么

- 按主题、异议或功能请求搜索客户通话。
- 找到 AI 打断了通话者、说得太慢或自我重复的通话。
- 将每个最终客户限定在他们自己的通话范围内。
- 在你的通话归档之上构建合规审查、QA 工作流或知识库。

## 核心模型

资产携带来自你自己系统的 ID 和范围。

| 字段 | 含义 |
| --- | --- |
| `asset_id` | 录音在你系统中的 ID。必填。会在搜索结果中返回，便于回溯到你的系统。 |
| `asset_class` | 一个逻辑分组 —— 一个客户、一个 workspace、一个项目。限制搜索范围，使结果不会跨越边界泄漏。 |
| `types` | 要索引的内容：`audio`、`video`、`transcript`，或任意组合。 |
| `metadata` | 用于关联和展示的扁平键/值对。 |

## 生命周期

<Steps>
  <Step title="摄入">
    携带你的 `asset_id` 发送 URL、S3 key 或文件上传。API 会立即返回一个 `task_id`。
  </Step>
  <Step title="索引">
    轮询任务直到状态变为 `ready`。索引在后台运行。
  </Step>
  <Step title="搜索">
    使用自然语言查询，并限定到某个 `asset_class` 或明确的 `asset_ids`。结果包含 `start_ms` 和 `end_ms`。
  </Step>
</Steps>

## 数据与隐私

我们对你的媒体建立索引，并保留可搜索的表示：嵌入向量、转写和元数据。源文件流经处理流水线，在索引完成后不会被保留。

**你保留对媒体的所有权。我们仅存储用于搜索所需的索引数据。**

区域化处理：
- **美国客户** —— 在美国处理。
- **欧盟和英国客户** —— 在欧洲处理。
