## 我们存储什么

我们对你的录音建立索引，并保留可搜索的表示：嵌入向量、转写和元数据。源文件流经处理流水线，在索引完成后不会被保留。

**我们存储的内容：**
- 嵌入向量（你媒体的索引表示）
- 资产元数据（`asset_id`、`asset_class`、`types`、自定义 `metadata` 字段）
- 摄入记录和时间戳

**我们不存储的内容：**
- 处理后的源音频或视频文件
- 原始形式的通话内容
- 处理完成后传输层的请求数据

## 区域化处理

数据在你的账户所开通的区域进行处理。

| 客户所在区域 | 处理位置 |
| --- | --- |
| 美国 | 美国 |
| 欧盟/英国 | 欧洲 |

欧盟和英国账户在欧洲基础设施上处理。

如果你需要此处未列出的特定区域，请[联系我们](https://discord.sf-voice.sh)。

## 欧盟和英国的数据处理

对欧盟和英国客户：

- 数据在欧洲基础设施上处理。
- 我们作为数据处理者。你仍然是录音中任何个人数据的数据控制者。
- 你可以通过 API 删除已索引的资产。删除会移除存储的表示 —— 嵌入向量、转写和元数据。
- 数据处理协议（DPA）可应需提供。

## 删除资产

`DELETE /v1/assets/:id` 会删除该资产及其索引表示。底层媒体不会受影响，因为我们在处理后不会保留源文件。

使用 TypeScript SDK：
```ts
await client.deleteAsset("call_001");
```
