## 资产源自你的系统

摄入请求会携带你的 `asset_id`，因此搜索结果可以无需 ID 转换就回连到你的数据库、CRM、客服工单、可观测性平台或通话日志。

```ts
await client.ingest({
  source: "url",
  asset_id: "call_8294",
  asset_class: "customer_acme",
  url: "https://storage.example.com/calls/8294.mp3",
});
```

## 资产分类定义搜索范围

`asset_class` 用于划定一同被搜索的边界。对大多数产品来说，它对应你的客户所习惯的实体：

- 一个最终客户
- 一个 workspace
- 一个项目
- 一个客服队列

<Warning>
  如果你在构建面向客户的搜索，请通过 `asset_class` 或显式的 `asset_ids` 进行限定。全局搜索（`scope: "all"`）会在你所有已索引的资产中搜索。
</Warning>

## 索引表面类型

`types` 控制哪些内容被索引和搜索。

| 类型 | 适用场景 |
| --- | --- |
| `audio` | 说话人身份、声学事件、音频层面的模式。 |
| `video` | 屏幕内容、视觉上下文、屏幕录制。 |
| `transcript` | 口语词汇、主题检索、对话语义搜索。 |

大多数语音 AI 场景三者都需要：

```ts
types: ["audio", "video", "transcript"]
```

## 索引是异步的

摄入会立即返回 `task_id`。实际的索引在后台执行。轮询直到状态变为 `ready` 或 `failed`。

```ts
const { task_id } = await client.ingest(request);
const task = await client.pollTask(task_id);

if (task.status === "failed") {
  throw new Error(task.error ?? "indexing failed");
}
```

任务状态：`pending` → `indexing` → `ready`（或 `failed`）。

## 搜索返回带时间戳的匹配项

结果包含 `start_ms` 和 `end_ms`：源录音中的毫秒偏移。

```json
{
  "asset_id": "call_8294",
  "score": 0.87,
  "start_ms": 42000,
  "end_ms": 58000,
  "match_type": "transcript"
}
```

`threshold` 控制匹配的严格程度。默认值为 `0.5`。提高它可以获得更少但置信度更高的结果。

## 你的数据归你所有

API 返回：`asset_id`、`asset_class`、`types`、分数和时间戳。供应商 ID、索引 key 以及流水线细节都保留在 API 之后，不会暴露。
