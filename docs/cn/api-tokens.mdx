API 令牌用于对发往 `/api/v1/*` 的每个请求进行身份验证 —— 这也是 SDK 在底层调用的端点。每个令牌都仅限于单个组织,因此你可以为生产、预发、CI 或团队成员分别签发令牌,而不必让出整个组织的控制权。

## 何时需要创建令牌

在以下场景下创建令牌:

- 在新的环境里运行[快速开始](/quickstart)或任意一个 [SDK](/sdks/overview)。
- 把媒体摄取接入后端服务、worker 或 CI 任务。
- 把受限的凭据交给同事或第三方工具。

你可以创建任意数量的令牌,撤销其中任何一个都不会影响其他令牌。

## 创建第一个令牌

当你首次进入一个尚未拥有任何令牌、也没有摄取过媒体的组织时,控制台会弹出一个三步的**欢迎**对话框:

<Steps>
  <Step title="创建令牌">
    点击**创建令牌**。系统会签发一个标签为 `default` 的令牌,并在对话框中展示出来。
  </Step>
  <Step title="安装 SDK">
    复制 `pnpm add @sf-voice/media` 命令,在你的项目里执行。
  </Step>
  <Step title="摄取你的第一个录音">
    复制初始代码片段,把令牌粘贴到 `SF_VOICE_TOKEN`,然后运行。
  </Step>
</Steps>

<Warning>
  完整的令牌**只会出现一次**。在关闭对话框之前,把它复制到密码管理器或密钥管理服务里 —— 之后无法再次获取。
</Warning>

只要组织里存在至少一个有效令牌,这个引导对话框就不会再出现。

## 在设置里管理令牌

打开**设置 → API 令牌**,随时可以创建更多令牌或撤销已有令牌。

### 创建令牌

1. 输入一个能说明用途的标签(例如 `production`、`ci` 或 `analytics-worker`)。
2. 点击**创建令牌**。
3. 从展示对话框中复制完整令牌,并安全保存。点击**我已保存**关闭对话框。

关闭后,控制台只会显示令牌前缀(例如 `sk_live_a1b2c3d4…`)、创建时间和最近一次使用时间。

### 撤销令牌

点击任意令牌旁的**撤销**,即可立即使其失效。正在进行的使用该令牌的请求会立即开始返回 `unauthorized`。撤销操作不可恢复 —— 如果需要轮换,请创建一个新令牌。

### 已过期的令牌

过期时间在当前时间之前的令牌会被标记为**已过期**,并停止认证请求。它们会保留在列表里以便审计,直到你将其撤销。

## 使用令牌

把令牌设置为环境变量,然后传给 SDK 客户端。永远不要把令牌提交到代码仓库。

```ts
import { SfVoiceMedia } from "@sf-voice/media";

const client = new SfVoiceMedia({
  baseUrl: "https://api.sf-voice.com",
  apiKey: process.env.SF_VOICE_API_KEY!,
});
```

如果你直接调用 REST API,请在 `X-API-Key` 请求头中传入令牌:

```sh
curl https://api.sf-voice.com/api/v1/assets \
  -H "X-API-Key: $SF_VOICE_API_KEY"
```

无效或已撤销的令牌会返回错误码 `unauthorized`。完整错误表请参见 [SDK 概览](/sdks/overview)。

## 最佳实践

- **每个环境一个令牌。** 把生产环境与预发、CI 分开,这样轮换一个不会影响其他环境。
- **认真标注令牌。** 当只剩前缀可见时,标签是区分令牌的唯一方式。
- **人员变动时轮换。** 团队成员离开或机器下线时,撤销并重新签发令牌。
- **保存到密钥管理器。** 把令牌当作密码对待 —— 不要打印日志、提交代码或粘贴到公共聊天中。
