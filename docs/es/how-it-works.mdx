## Los assets nacen en su sistema

Las peticiones de ingesta llevan su `asset_id`, de modo que los resultados de búsqueda pueden enlazar con su base de datos, CRM, ticket de soporte, plataforma de observabilidad o registro de llamadas sin traducción de IDs.

```ts
await client.ingest({
  source: "url",
  asset_id: "call_8294",
  asset_class: "customer_acme",
  url: "https://storage.example.com/calls/8294.mp3",
});
```

## Las asset classes definen el alcance de la búsqueda

`asset_class` es cómo traza la frontera de lo que se busca conjuntamente. Para la mayoría de productos, se corresponde con la entidad en la que piensan sus clientes:

- un cliente final
- un workspace
- un proyecto
- una cola de soporte

<Warning>
  Si está construyendo búsqueda orientada al cliente, limite por `asset_class` o `asset_ids` explícitos. La búsqueda global (`scope: "all"`) abarca todos sus assets indexados.
</Warning>

## Tipos de superficies de indexación

`types` controla qué se indexa y qué se busca.

| Tipo | Úselo cuando |
| --- | --- |
| `audio` | Identidad del hablante, eventos acústicos, patrones a nivel de sonido. |
| `video` | Contenido en pantalla, contexto visual, grabaciones de pantalla. |
| `transcript` | Palabras habladas, recuperación temática, búsqueda semántica sobre diálogo. |

La mayoría de casos de uso de IA de voz quieren los tres:

```ts
types: ["audio", "video", "transcript"]
```

## La indexación es asíncrona

La ingesta devuelve un `task_id` de inmediato. La indexación real ocurre en segundo plano. Sondee hasta `ready` o `failed`.

```ts
const { task_id } = await client.ingest(request);
const task = await client.pollTask(task_id);

if (task.status === "failed") {
  throw new Error(task.error ?? "indexing failed");
}
```

Estados de tarea: `pending` → `indexing` → `ready` (o `failed`).

## La búsqueda devuelve coincidencias con marcas de tiempo

Los resultados incluyen `start_ms` y `end_ms`: desplazamientos en milisegundos dentro de la grabación fuente.

```json
{
  "asset_id": "call_8294",
  "score": 0.87,
  "start_ms": 42000,
  "end_ms": 58000,
  "match_type": "transcript"
}
```

`threshold` controla qué tan estricta debe ser la coincidencia. El valor por defecto es `0.5`. Súbalo para obtener menos resultados, con mayor confianza.

## Sus datos siguen siendo suyos

La API devuelve: `asset_id`, `asset_class`, `types`, puntuaciones y marcas de tiempo. Los IDs de proveedores, claves de índice y detalles del pipeline quedan detrás de la API.
