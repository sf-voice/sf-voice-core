San Francisco Voice Company crea APIs y SDKs de búsqueda de medios para productos de IA de voz.

Nuestra primera funcionalidad es un motor de búsqueda para medios. Ingiera audio o vídeo, búsquelo con lenguaje natural y use las marcas de tiempo devueltas para saltar de nuevo a la grabación. No necesita construir antes su propio almacenamiento de transcripciones, pipeline de embeddings o capa de búsqueda de medios.

<Warning>
  Esta documentación cubre la versión `0.1.1`. Estamos iterando rápido, así que fije la versión del SDK y espere cambios incompatibles entre versiones hasta que la API se estabilice.
</Warning>

La búsqueda es donde empezamos. Lo siguiente son herramientas de benchmarking y depuración para sistemas de IA de voz.

## Lo que puede construir

- Buscar llamadas de clientes por tema, objeción o petición de funcionalidad.
- Encontrar llamadas en las que su IA interrumpió al interlocutor, habló demasiado lento o se repitió.
- Limitar a cada cliente final a sus propias llamadas.
- Construir revisión de cumplimiento, flujos de QA o bases de conocimiento sobre su archivo de llamadas.

## El modelo central

Los assets llevan los IDs y el alcance desde su propio sistema.

| Campo | Qué es |
| --- | --- |
| `asset_id` | Su ID para la grabación. Requerido. Se devuelve en los resultados de búsqueda para que pueda enlazar de vuelta con su sistema. |
| `asset_class` | Un grupo lógico — un cliente, un workspace, un proyecto. Limita la búsqueda para que los resultados no se filtren entre fronteras. |
| `types` | Qué indexar: `audio`, `video`, `transcript`, o cualquier combinación. |
| `metadata` | Pares clave/valor planos para correlación y visualización. |

## Ciclo de vida

<Steps>
  <Step title="Ingerir">
    Envíe una URL, una clave de S3 o una carga de archivo con su `asset_id`. La API devuelve inmediatamente un `task_id`.
  </Step>
  <Step title="Indexar">
    Sondee la tarea hasta que alcance `ready`. La indexación se ejecuta en segundo plano.
  </Step>
  <Step title="Buscar">
    Consulte con lenguaje natural, limitado a una `asset_class` o `asset_ids` explícitos. Los resultados incluyen `start_ms` y `end_ms`.
  </Step>
</Steps>

## Datos y privacidad

Indexamos sus medios y conservamos la representación buscable: embeddings, transcripciones y metadatos. Los archivos fuente pasan por el pipeline de procesamiento y no se retienen una vez completada la indexación.

**Usted mantiene la propiedad de sus medios. Nosotros almacenamos los datos indexados necesarios para la búsqueda.**

Procesamiento regional:
- **Clientes de EE. UU.** — procesados en Estados Unidos.
- **Clientes de UE y Reino Unido** — procesados en Europa.
