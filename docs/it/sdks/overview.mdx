## Autenticazione

Gli SDK inviano la tua chiave API come header di richiesta `X-API-Key`. Conservala in una variabile d'ambiente ed evita di committarla nel controllo versione.

## Gestione degli errori

Gli SDK usano tre categorie di errori:

| Errore | Quando viene lanciato |
| --- | --- |
| Errore API | Il server ha restituito una risposta non-2xx. Porta `code`, `message` e `status`. |
| Timeout di richiesta | Una singola richiesta HTTP ha superato il timeout configurato. |
| Timeout di polling | `pollTask` ha esaurito il tempo prima che il task raggiungesse `ready` o `failed`. |

Codici di errore dell'API:

| Codice | Significato |
| --- | --- |
| `unauthorized` | Chiave API non valida o mancante. |
| `not_found` | L'ID dell'asset o del task non esiste. |
| `bucket_not_connected` | Sorgente S3 usata prima della configurazione del bucket. |
| `s3_access_denied` | L'SDK non puĆ² leggere dalla chiave S3 specificata. |
| `unsupported_format` | Il tipo di file non ĆØ supportato per l'indicizzazione. |
| `file_too_large` | L'upload supera il limite di dimensione. |
| `rate_limited` | Troppe richieste. Applica back-off e riprova. |
