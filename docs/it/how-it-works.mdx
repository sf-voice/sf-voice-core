## Gli asset nascono nel tuo sistema

Le richieste di ingestione portano il tuo `asset_id`, così i risultati di ricerca possono ricollegarsi al tuo database, CRM, ticket di supporto, piattaforma di osservabilità o registro chiamate senza traduzione di ID.

```ts
await client.ingest({
  source: "url",
  asset_id: "call_8294",
  asset_class: "customer_acme",
  url: "https://storage.example.com/calls/8294.mp3",
});
```

## Le asset class definiscono l'ambito della ricerca

`asset_class` è il modo in cui tracci il confine di ciò che viene cercato insieme. Per la maggior parte dei prodotti, corrisponde all'entità in cui ragionano i tuoi clienti:

- un cliente finale
- un workspace
- un progetto
- una coda di supporto

<Warning>
  Se stai costruendo ricerca rivolta ai clienti, limita per `asset_class` o `asset_ids` espliciti. La ricerca globale (`scope: "all"`) attraversa tutti i tuoi asset indicizzati.
</Warning>

## Tipi di superfici di indicizzazione

`types` controlla cosa viene indicizzato e cercato.

| Tipo | Usalo quando |
| --- | --- |
| `audio` | Identità del parlante, eventi acustici, pattern a livello sonoro. |
| `video` | Contenuto a schermo, contesto visivo, registrazioni dello schermo. |
| `transcript` | Parole pronunciate, recupero per argomento, ricerca semantica sui dialoghi. |

La maggior parte dei casi d'uso di IA vocale vuole tutti e tre:

```ts
types: ["audio", "video", "transcript"]
```

## L'indicizzazione è asincrona

L'ingestione restituisce immediatamente un `task_id`. L'indicizzazione vera e propria avviene in background. Esegui il polling fino a `ready` o `failed`.

```ts
const { task_id } = await client.ingest(request);
const task = await client.pollTask(task_id);

if (task.status === "failed") {
  throw new Error(task.error ?? "indexing failed");
}
```

Stati del task: `pending` → `indexing` → `ready` (o `failed`).

## La ricerca restituisce corrispondenze con timestamp

I risultati includono `start_ms` e `end_ms`: offset in millisecondi nella registrazione sorgente.

```json
{
  "asset_id": "call_8294",
  "score": 0.87,
  "start_ms": 42000,
  "end_ms": 58000,
  "match_type": "transcript"
}
```

`threshold` controlla quanto deve essere stringente la corrispondenza. Il valore predefinito è `0.5`. Aumentalo per ottenere meno risultati, ma con maggiore confidenza.

## I tuoi dati restano tuoi

L'API restituisce: `asset_id`, `asset_class`, `types`, punteggi e timestamp. Gli ID dei fornitori, le chiavi di indice e i dettagli della pipeline restano dietro l'API.
